# Binance USDS-M Futures Trend v0.9 Historical Backtest Framework Plan

Created: UTC 2026-06-15 13:24:50 / 北京时间(UTC+8) 2026-06-15 21:24:50

## Goal

Add a paper-only historical backtest framework for the existing >=1h Binance USDS-M trend strategy, so we can evaluate CAGR / drawdown / risk-adjusted metrics before any live or testnet execution work.

## Scope

- Keep all strategy execution `paper only`; no signed endpoints and no live orders.
- Use free Binance USDS-M K-line data via existing `/fapi/v1/klines` fetch path.
- Reject `<1h` intervals via existing interval validation.
- Backtest long-only trend participation generated by the existing EMA50/EMA200 + ATR decision logic.
- Output metrics:
  - CAGR;
  - max drawdown;
  - Calmar;
  - Sharpe;
  - win rate;
  - average holding time in candles;
  - turnover;
  - per-symbol contribution.
- Add CLI `--backtest` mode for single symbol, selected symbols, or full configured universe.

## Intended Test Interface

1. `test_backtest_symbol_outputs_paper_metrics_with_timezone_labels`
   - Build synthetic >=240 candle fixture.

   - Assert paper-only output, UTC / 北京时间（UTC+8） timestamps, required metrics, equity curve, per-symbol contribution, no live-order fields.

2. `test_backtest_rejects_short_interval_and_insufficient_history`
   - Assert `30m` rejected.
   - Assert fewer than 201 candles rejected.

3. `test_backtest_symbols_aggregates_per_symbol_contribution_and_errors`
   - Monkeypatch `fetch_klines` for BTC/SOL/ETH fixtures.
   - Call `trend.backtest_symbols([...], interval="4h", include_symbols_with_errors=True)`.
   - Assert aggregate metrics, per-symbol results/contribution, and paper-only mode.

4. `test_main_can_emit_backtest_json_without_context_or_orders`
   - Monkeypatch `fetch_klines`.
   - Call `trend.main(["--backtest", "--symbols", "BTCUSDT", "--interval", "1h", "--limit", "240"])`.
   - Assert JSON contains `ok=true`, `backtest`, `paper only`, UTC / 北京时间（UTC+8）, and no order/execution keys.

## Risks / Guardrails

- v0.9 is a framework, not final strategy proof. Do not present backtest output as live returns.
- Initial simulation is intentionally conservative and simple; strategy refinement belongs to v1.0.
- No short periods, no paid APIs, no API keys, no real orders.
